## 1. Deploy Commands

```bash
# 1. Push the already-fixed source (triggers Render Blueprint build)
git push origin main          # backend bin builds via cargo; dashboard via npm

# 2. In Render dashboard: New → Blueprint → connect iamtemamlight/ms1
#    Render reads render.yaml and creates:
#      - allbright-backend   (Docker, backend/Dockerfile, /healthz)
#      - allbright-dashboard(Docker, ./Dockerfile, /api/health)
#      - allbright-redis    (redis)
#      - allbright-db        (postgres)

# 3. Fill the sync:false secrets when prompted (see §3 Config Table).

# 4. Verify backend health (replace <backend>.onrender.com):
curl -f https://<backend>.onrender.com/healthz        # -> "ok" (auth-exempt)
curl  https://<backend>.onrender.com/api/metrics | head
curl  https://<backend>.onrender.com/api/governance/cards

# 5. Verify dashboard proxy reaches backend:
curl -f https://<dashboard>.onrender.com/api/health   # proxied to backend

# 6. Dashboard opens at https://<dashboard>.onrender.com
```

Render auto-wires `RUST_BACKEND_URL` (dashboard) and matching `API_KEY`/`RUST_API_KEY` between the two services.

## 4. Wallet System (Execution vs Withdrawal)

- **Execution Wallet** — single, backend-owned. `PRIVATE_KEY` lives only in Render env
  (AES-256-GCM vault). Frontend shows it read-only (`accumulatedProfitsUsd`, address,
  network) with a "Vault-managed · key never exposed" badge. No key input anywhere.
- **Withdrawal Wallets** — user-managed list, frontend-local (`allbright_wallets` in
  localStorage). Stored fields limited to **name, address, chain, created_date, status**.
  The `CustomWalletItem.privateKey` field was removed this session to enforce the
  security requirement (no PRIVATE_KEY / SEED / MNEMONIC on frontend).
- **Deposit / Withdraw / Transfer** call the backend `compat_*` routes (verified live).
- **Backend note:** `/api/wallet/deposit|withdraw` currently update the *execution* wallet
  `totalValueUsd`, not a specific withdrawal-wallet row. If per-withdrawal-wallet balances
  are needed, add a `wallet_id` param to those routes (follow-up, not blocking).

## 5. Dashboard View

- Serves from `./Dockerfile` → Express `server.cjs` proxies `/api/*` to backend.
- Polls every 3 s: `/api/metrics`, `/api/opportunities`, `/api/settings`,
  `/api/wallet`, `/api/governance/cards` (`Promise.all`, null-safe).
- Metric cards now read **real** `compat_metrics` fields (`totalProfitUsd`,
  `successfulTradesCount`, `avgGasCostUsd`, `scanLatencyMs`, `mevAttackPct`) and a
  **populated 7-point `profitTrend`** (previously empty).
- Opportunity table driven by `/api/opportunities` (derived from profit metrics).
- Health banner clears once `RUST_BACKEND_URL` resolves and `/api/health` returns ok.

## 7. Go-Live Sequence (recommended)

1. Rotate all exposed secrets (Neon, RPC, AI keys, wallet key).
2. Apply Render Blueprint; set env vars from §2.
3. Keep `PILOT_MODE=true` + `RUST_API_KEY`/`API_KEY` matched; deploy.
4. Smoke-test §1 curls; confirm dashboard banner clears.
5. Run 24 h **pilot** (paper/simulation); watch gas & P99 latency KPIs.
6. Flip `PILOT_MODE=false` only after pilot is green and external audit signed off.
7. Enable `AUTO_TRANSFER_ENABLED` per treasury policy.
